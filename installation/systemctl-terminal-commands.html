Systemctl Reference : Terminal Commands

In this article we will learn some basic commonly used systemctl terminal commands.

Having some familiarity with these commands will help you configure and automate process execution using systemd.

Prerequisite Knowledge

To understand this guide you dont need to be an expert but having some basic familiarity with linux terminal commands will help you.

Let us begin the step by step guide to learn systemctl commands

Enable a service

                     Code
                  
    sudo systemctl enable service.service

Enables a service. When you enable a service the service is added to the list of services which will start automatically at the boot time.

Check whether the service is enabled or not

                     Code
                  
    sudo systemctl is-enabled service.service

This command checks whether the service is enabled or not

Disable's a service

                     Code
                  
    sudo systemctl disable service.service

This command will disable the service with the given name and remove it from the list of service which will start at the boot time.

Reload the services

                     Code
                  
    sudo systemctl daemon-reload

Reloads the systemctl unit files. This command should be executed after any change in unit files are made

Restart the service

                     Code
                  
    sudo systemctl restart service.service

Restart the service with the given name

Get the status of the service

                     Code
                  
    sudo systemctl status service.service

Gets the current status of the service with the given name

Create a unit file | Create a text file

                     Code
                  
    sudo touch filename.service

creates a text file

Make a copy of a file

                     Code
                  
    sudo cp filename newfilename

Make a copy of the file

Edit a file with a text editor

                     Code
                  
    sudo nano filename

opens a file for editing in the nano text editor

Restart server from command line

                     Code
                  
    shutdown -r now

Restart the server using command line

Get the process ID for the service

                     Code
                  
    ps -ef | grep service

Get the process id for the service with the given name

Kill a process

                     Code
                  
    sudo kill -9 process_number

Kill the process with the given process id

Switch to the directory which contains unit files

                     Code
                  
    cd /etc/systemd/system

change directory to where all the unit files are located

Edit a unit file

                     Code
                  
    sudo systemctl edit service.service

Opens the unit file with the given name in a text editor

An Example of a unit file

                     Code
                  
    [Unit]

    Description=nearbyshops_sds: Nearbyshops Service Discovery Service

    After=syslog.target network.target

    [Service]

    WorkingDirectory=/nearbyshops/sds
    ExecStart=/usr/lib/jvm/java-8-openjdk-amd64/bin/java /nearbyshops/nearby-shops-api.jar
    Type=simple

    Restart=always

    [Install]

    WantedBy=multi-user.target

This is an example of a unit file which is used to automate the execution of a java jar file
