Location Concepts

In this article we will study the basic concepts of location.

Prerequisite Knowledge

To understand this upgrade guide you dont need to be an expert but having some basic understanding of software and database will help you.

Nearby Shops is a location based app. The end-user is supposed to see the shops which are near to him / her. The shops are supposed to be filtered by delivery range or proximity. We will study few concepts which will help you understand how location works !

Latitude and Longitude

Latitude and longitude are the location coordinates used to identify any location on the Earth. We use GPS in the smart-phone to obtain latitude and longitude. You can study more about location coordinates in this wikipedia article. https://en.wikipedia.org/wiki/Geographic_coordinate_system

haversine formula

We use haversine formula to calculate distance between two locations. You dont need to know how it works in order to use it. But still if you're interested you can learn more about it. There are so many good articles available on the internet.

Delivery Range

Each shop has a limited delivery range and it can deliver only upto certain distance from its physical location. The end-user is supposed to see only those shops-items which can be delivered at his / her location. If the end-user is seeing the items that cannot be delivered at his location then our concept of local shopping will simply does not work ! Therefore we need the ability to filter shop-items by delivery range.

We can define delivery range filter in the SQL Query. 

                     Code
                  
    distance (calculated using haversine formula) < shop.delivery_range

Proximity

Proximity and delivery range are not the same thing. A shop can be near to you but it may still not be able to deliver to your location because it may have very low delivery range - an area upto which it can deliver. Another shop can be far away from you but it can still have the ability to deliver at your location because it has higher delivery range.

We can see that proximity and delivery range are not the same and they are different concepts. Filtering shop-items based on proximity is required when home delivery is not provided. There can be shops which does not provide home delivery and require the end-user to pick the order from the shop. In this situation we dont need to filter shop-items using delivery range. An end-user will be more interested to see shop-items and their distance from his / her current location.

We can define proximity filter in the SQL Query. 

                     Code
                  
    distance (calculated using haversine formula) < proximity_value
